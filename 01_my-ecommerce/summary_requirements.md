# สรุปข้อมูลโครงการ BedSpace E-Commerce (Requirement Summary)

โครงการนี้เป็นระบบร้านค้าออนไลน์สำหรับเฟอร์นิเจอร์และอุปกรณ์ที่ช่วยให้การทำงานหรือเรียนบนเตียงถูกสุขลักษณะและลดอาการปวดหลัง

## 1. ข้อมูลธุรกิจ (Business Concept)

- **ชื่อระบบ:** BedSpace E-Commerce System
- **ปัญหา (Problem Statement):** ผู้ที่นั่งทำงานหรือเรียนบนเตียงมักมีอาการตัวไหล ปวดหลัง และหาโต๊ะที่พอดีกับเตียงยาก
- **โซลูชัน (Solution):** จำหน่ายอุปกรณ์ที่ช่วยซัพพอร์ตสรีระและโต๊ะที่ออกแบบมาเพื่อการใช้งานบนเตียงโดยเฉพาะ
- **สินค้าหลัก:**
  - โต๊ะข้างเตียงปรับระดับได้
  - โต๊ะคร่อมเตียง (Over-bed Desk)
  - โต๊ะพับสำหรับตั้งบนเตียง
  - เบาะรองหลังซัพพอร์ตสรีระ

## 2. ตัวอย่างกลุ่มเป้าหมาย (Target Audience / Persona)

- **Persona:** "นัตตี้" (Nutty) อายุ 22 ปี นักศึกษา/คนทำงานแบบ Remote (WFH)
- **ไลฟ์สไตล์:** ชอบใช้เวลาในห้องนอน, ช้อปปิ้งออนไลน์เป็นประจำ, ไม่พกเงินสด (เน้นจ่ายผ่านระบบ)
- **เป้าหมาย:** ต้องการทำงานบนเตียงได้อย่างสบายและถูกหลักสรีรศาสตร์

## 3. ความต้องการของผู้ใช้ (User Requirements / Product Backlog)

### สำหรับลูกค้า (Customer)

- **การเลือกซื้อ:** สามารถดูรายการสินค้า, ดูรายละเอียดสินค้า (ราคา/สต็อก) และหยิบใส่ตะกร้าได้
- **การสั่งซื้อ:** สั่งซื้อสินค้าโดยระบุข้อมูลติดต่อ (ชื่อ, อีเมล, เบอร์โทรศัพท์) และชำระเงินผ่านระบบ
- **การช่วยเหลือ:** สามารถเปิด Ticket (ติดต่อสอบถาม) เพื่อขอความช่วยเหลือจากแอดมินได้

### สำหรับผู้ดูแลระบบ (Admin)

- **จัดการสินค้า:** เพิ่ม, แก้ไข หรือลบข้อมูลสินค้าในระบบได้
- **จัดการคำสั่งซื้อ:** ดูรายการสั่งซื้อและอัปเดตสถานะ (เช่น Paid, Pending, Shipped)
- **จัดการการช่วยเหลือ:** อ่านและตอบกลับ Support Ticket ของลูกค้า

## 4. โครงสร้างข้อมูล (Data Structure)

ระบบมีการจัดเก็บข้อมูลหลัก 5 ส่วน:

1. **Customers:** ข้อมูลลูกค้า (ชื่อ, อายุ, อาชีพ, อีเมล, เบอร์โทร)
2. **Products:** ข้อมูลสินค้า (ชื่อ, ราคา, สต็อก)
3. **Orders:** ข้อมูลการสั่งซื้อและสถานะ
4. **Order Items:** รายละเอียดสินค้าในแต่ละคำสั่งซื้อ (จำนวน)
5. **Support Tickets:** ข้อมูลการแจ้งปัญหาหรือสอบถามจากลูกค้า

## 5. รูปแบบการทำงานด้านเทคนิค (Technical Context)

- **Frontend:** เว็บไซต์ร้านค้าสำหรับลูกค้าเลือกซื้อและแอดมินจัดการระบบ
- **Backend/Database:** ใช้ PostgreSQL ในการเก็บข้อมูล และมีการจำลอง Logic เบื้องต้นด้วย JavaScript
- **Payment:** รองรับระบบ Payment Gateway (Cashless)
