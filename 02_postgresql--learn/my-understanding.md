### PostgreSQL for Web Developer
##### Type of Database
- Relational -- เก็บข้อมูลในรูปแบบตารางที่มีความสัมพันธ์ สามารถเชื่อมโยงกันข้ามตารางได้ (PostgreSQL ใช้จัดการกับฐานข้อมูลประเภทนี้)
- Document-based 
##### **Table, Row และ Column** สัมพันธ์กันอย่างไร?
	Table เป็นเซ็ตที่ใหญ่ที่สุด ประกอบด้วย ลำดับถัดมาคือ Column ที่กำหนดค่าไว้ชัดเจนว่าต้องการข้อมูลอะไรในรูปแบบการเก็บแบบใด โดยค่าที่จะเก็บเข้าไปก็จะเพิ่มมาเรื่อย ๆ ในลักษณะของ Row หรือเรคคอร์ด
##### Type of Data
- มีหน้าที่เก็บข้อมูลที่มีรูปแบบต่างกัน ขึ้นอยู่กับการนำไปใช้ของแต่ละข้อมูล เช่น
	- `INT` เก็บค่าเป็นตัวเลข สามารถนำไปคำนวณค่าต่อได้
	- `VARCHAR` เก็บค่าเป็น string ใช้อธิบายสิ่งต่าง ๆ ได้ดี
	- `BOOLEAN` เก็บค่าแค่จริงหรือเท็จ ก็เอาไปใช้ในการตัดสินใจต่อ ๆ ไปได้
	- `TIMESTAMP` เก็บค่าเป็นวันที่ ก็จะทำให้เราสามารถเรียงลำดับข้อมูลตามที่ต้องการได้ดียิ่งขึ้น
- ทั้งหมดทั้งมวลขึ้นอยู่กับความต้องการของผู้ใช้งาน ว่าจะเลือกใช้ต่างกันอย่างไร 
- นอกจากนี้ยังมีเรื่องของขนาดข้อมูลที่เก็บ แต่ละชนิดก็มีขนาดต่างกัน ซึ่งน่าจะส่งผลต่อความเร็วของฐานข้อมูลเช่นกัน

> **CRUD = Create-Read-Update-Delete**
> 	- คือ 4 ฟังก์ชันพื้นฐานที่ใช้ในการจัดการข้อมูลในระบบฐานข้อมูลหรือแอปพลิเคชัน

##### **Primary Key** คืออะไร? ทำไมทุก Table ควรจะมีมัน?
	คือคอลัมน์ ๆ หนึ่งของตาราง ที่เก็บค่าเป็น Unique value ทุกตารางควรมีเพื่อความเร็วและความถูกต้องในการหาข้อมูล ตารางที่ไม่มี key มีความเสี่ยงสูงที่จะเอาไป join กับ ตารางอื่นแล้วได้ผลลัพธ์ไม่ตรงตามต้องการ เช่น ข้อมูลซ้ำซ้อน หรือ ข้อมูลขาดหาย

> **ER Diagram** สำคัญมาก ในการสร้าง DB
> 	- การเขียน ER เส้นเชื่อมต้องเชื่อมค่าเดียวกัน
> 	- สามารถระบุ 1 / * ไว้ที่หัวเส้นแต่ละด้านได้ เพื่อระบุว่า ค่าแต่ละตารางมีได้กี่เรคคอร์ด 
> 	- PK คือ Unique ของตารางนั้น ๆ (อาจไปเป็น FK ในตารางอื่น ๆ ได้)

##### หลักการเขียน query
- เขียนให้เป๊ะที่สุด เอาที่ต้องการเท่านั้น
##### Typical SQL Query Pattern
- SELECT -- เลือก **คอลัมน์** ที่ต้องการ (หรือใช้ `*` เพื่อเลือกทั้งหมด)
- FROM -- ระบุ **ชื่อตาราง** ที่เป็นต้นทางของข้อมูล
- WHERE -- ใช้ **กรองแถว** (Rows) ตามเงื่อนไขที่กำหนด (ทำก่อนการจัดกลุ่ม *ห้ามใช้ฟังก์ชันคำนวณที่นี่*)
- GROUP BY -- ใช้ **จัดกลุ่ม** ข้อมูลที่มีค่าเหมือนกัน (มักใช้คู่กับฟังก์ชันคำนวณ เช่น `SUM`, `COUNT`, `AVG`)
- HAVING -- ใช้ **กรองข้อมูลหลังการจัดกลุ่ม** (ใช้คู่กับ Aggregate Functions เช่น `HAVING COUNT(*) > 1`) *ใช้คู่กับ GROUP BY เพื่อกรองพวกค่า SUM หรือ COUNT // 99% ใช้คู่กันเพื่อให้ได้ผลลัพธ์ตามกลุ่มที่ต้องการ*
- ORDER BY -- ใช้ **เรียงลำดับ** ผลลัพธ์ (เช่น `ASC` น้อยไปมาก, `DESC` มากไปน้อย)

##### ลำดับการทำงานของคอมในการรัน SQL (SQL Order of Execution)
```
FROM + JOIN
↓
WHERE
↓
GROUP BY
↓
HAVING
↓
SELECT
↓
ORDER BY
↓
LIMIT
```

**คำสั่ง SQL ในพาร์ทการสร้างและอัปเดตข้อมูล**
>	- CREATE TABLE -- สร้างตาราง
>	- ALTER TABLE -- เปลี่ยนโครงสร้างตาราง (เช่น เพิ่ม Column ใหม่ หรือเปลี่ยน Data Type)
>	- INSERT -- เพิ่มข้อมูลลงตาราง
>	- UPDATE -- เปลี่ยนค่าข้อมูลที่มีอยู่แล้วในตาราง 
>	- DELETE -- ลบข้อมูลในตาราง 
>	**ข้อควรระวังของ UPDATE และ DELETE: ถ้าลืมใส่ `WHERE` ในคำสั่ง `UPDATE` หรือ `DELETE` จะส่งผลกับ **"ทุกแถว"** ในตารางทันที**
##### Join Concepts
- **INNER JOIN** -- Fetch Only Matching Records (*If you write just JOIN, it defaults to INNER JOIN.*)
	- ใช้เมื่อ -- ต้องการข้อมูลที่ Intersect กัน (มีเรคคอร์ดในทั้งสองตาราง)
- **LEFT JOIN** 
	- ใช้เมื่อ -- ต้องการทุกเรคคอร์ดของตารางหลัก
- **RIGHT JOIN** 
	- ใช้เมื่อ -- ต้องการทุกเรคคอร์ดของตารางรอง
- **FULL OUTER JOIN**
	- ใช้เมื่อ -- ต้องการข้อมูลที่ Union กัน (เอาทุกเรคคอร์ดจากท้งสองตาราง)

##### SQL Normalization
    **SQL Normalization** (หรือการทำให้เป็นบรรทัดฐาน) คือกระบวนการ **ออกแบบฐานข้อมูล** เพื่อลดความซ้ำซ้อนของข้อมูล (Data Redundancy) และป้องกันปัญหาความผิดพลาดในการจัดการข้อมูล (Data Anomalies) เช่น การเพิ่ม ลบ หรือแก้ไขข้อมูลแล้วเกิดการขัดแย้งกันเอง
        - **1NF** (First Normal Form) -- Remove **repeating groups**. (atomic values only) *อย่าเก็บหลายค่าใน 1 ช่อง*
        - **2NF** (Second Normal Form) -- Remove **partial dependencies** *กฎเหล็ก: ต้องผ่าน 1NF มาก่อน และ "ทุก Column ที่ไม่ใช่ Key ต้องขึ้นอยู่กับ Primary Key 'ทั้งหมด' ไม่ใช่แค่บางส่วน" (in case many pk)*
        - **3NF** (Third Normal Form) -- Remove **transitive dependencies** *กฎเหล็ก: ต้องผ่าน 2NF มาก่อน และ "Column ที่ไม่ใช่ Key ห้ามขึ้นตรงต่อกันเป็นทอด ๆ" (ต้องขึ้นตรงต่อ Primary Key เท่านั้น)*
