# **MongoDB**
    MongoDB เป็นฐานข้อมูลประเภท Document-based มีโครงสร้างแบบ NoSQL

#### **เทียบ RDBS vs DDBS**

| **คุณสมบัติ**          | **RDBMS (Relational Database)**                 | **Document-based Database (NoSQL)**                                          |
| ---------------------- | ----------------------------------------------- | ---------------------------------------------------------------------------- |
| **รูปแบบการจัดเก็บ**   | เก็บเป็น **ตาราง (Table)** ที่มีความสัมพันธ์กัน | เก็บเป็น **เอกสาร (Document)** รูปแบบคล้าย JSON/BSON                         |
| **ภาษาที่ใช้จัดการ**   | ใช้ SQL (Structured Query Language)             | ใช้ MQL (MongoDB Query Language)                                             |
| **โครงสร้างข้อมูล**    | Database → **Tables → Rows → Columns**          | Database → **Collections → Documents → Fields**                              |
| **โครงสร้าง (Schema)** | ตายตัว ต้องกำหนดคอลัมน์และชนิดข้อมูลล่วงหน้า    | ยืดหยุ่น (Dynamic Schema) เพิ่มฟิลด์ใหม่หรือทำข้อมูลซ้อนกัน (Nested) ได้ง่าย |
>- **DDBMS** เหมาะกับ **agile** มากกว่า เพราะว่ามุ่งเน้นในเรื่องของการยืดหยุ่น (flexible schema) และ ความน้อย
>- แต่ละ document จะมี unique id ซึ่ง MongoDB จะแจกจ่ายให้มาเอง

#### **MongoDB important concept**
- file format -- ถูกแปลงเป็น BSON ก่อนเข้า mgdb 

#### **BSON vs JSON**
> **BSON = JSON upgraded** *-- Binary Based vs Text based*

| ชนิดข้อมูลที่ BSON มีเพิ่มขึ้น | คำอธิบาย                                                                  |
| ------------------------------ | ------------------------------------------------------------------------- |
| **ObjectId**                   | รหัส 12-byte ที่ไม่ซ้ำกัน (ใช้เป็น `_id` หลักของเอกสาร)                   |
| **Date & Timestamp**           | เก็บข้อมูลวันที่และเวลา UTC ได้แม่นยำ                                     |
| **Binary Data**                | ใช้จัดเก็บข้อมูลประเภทไฟล์ หรือรูปภาพ                                     |
| **Regular Expression**         | เก็บรูปแบบ Pattern สำหรับใช้ค้นหาข้อความ (RegEx)                          |
| **JavaScript Code**            | เก็บชุดคำสั่งหรือฟังก์ชัน JavaScript ไว้ในฐานข้อมูลได้                    |
| **ตัวเลขแบบเฉพาะเจาะจง**       | แยกระดับความละเอียดได้ เช่น 32-bit Integer, 64-bit Integer และ Decimal128 |
| **MinKey / MaxKey**            | ค่าพิเศษที่ใช้เปรียบเทียบว่ามีค่าต่ำสุดหรือสูงสุดเสมอ                     |

### **MQL**
#### **find() & findOne()**
    การอ่านข้อมูลจะใช้คำสั่งหลักคือ find() หรือ findOne() โดยสามารถกำหนดเงื่อนไขการค้นหา (Query), เลือกแสดงเฉพาะบางฟิลด์ (Projection), และตั้งค่าเพิ่มเติมเช่นเรียงลำดับหรือจำกัดจำนวน (Options) ได้

#### **advanced query**
- **aggregation pipeline** -- การรวบรวมของหลาย ๆ อย่างให้เป็นนสิ่งเดียวกัน
- Mental Model: โรงงาน และสายพาน
- เป็นเครื่องมือจัดการข้อมูลขั้นสูงที่ทำงานคล้าย "สายพานโรงงาน" โดยให้ข้อมูลไหลผ่านสเตจ (Stages) ต่างๆ
- ทำให้ผมนึกถึงการทำงานของ power query ที่จัดการข้อมูลเป็นสเต็ป ๆ
